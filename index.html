tmux is a terminal multiplexer

Um, what?


Split the screen, like in your text editor.

Open multiple tabs like your browser.

Installing


			< brew install tmux

Install on windows with cygwin.

Usage*


			< tmux

Sessions


You now have your first "session"

"sessions" are like "tabs" in the browser

First, lets give it a name*


<control + b> + $

Names will be important when you have a dozen sessions.

Splitting the pane.



Horizontally Horizontally 
<control + b> + "
 
<control + b> + $

Switch focus*



<control + b> arrows

Kill the Pane*



<control + b> x, y

Other fun*


Next pane 
<control + b> + n
Prev pane 
<control + b> + p
Rotate panes 
<control + b> + <control + o>
Toggle layouts 
<control + b> + spacebar

Changing gears


Time to return to your first project.


<control + b> + s

Pros


  • Be more productive
  • Scriptable, really no limit to what you can do
  • Keeps a list of what you are working on lately

Cons


  • More key bindings
  • Difficult to make sessions persistent

What's next?


  • Scripting tmux's startup
  • Or get sessions to persist
